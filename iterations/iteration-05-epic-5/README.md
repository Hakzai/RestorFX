# Restaurant Management - Iteration 05 (EPIC 5)

## Goal

Deliver the first basic order/sales flow with persistence and a simple JavaFX screen.

## Scope

- Order and order item models
- Order persistence through repository/service layers
- Basic JavaFX order screen
- Order header + item persistence in SQLite

## Exit Criteria

- Orders can be created from the UI
- Order items are persisted with the order header
- Service validates item presence and quantity
- Repository can load recent orders and their items
